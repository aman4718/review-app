

MILESTONE 2
App Review Insights Analyser
Problem Statement, Architecture & Phase-wise Build Guide
Product: ChatGPT Voice for India
April 2026 | AI Product Management Cohort



1. Problem Statement
ChatGPT has over 800 million potential users in India — yet the product team has no systematic, automated way to understand what those users say about the app every week. Reviews are written in English, Hindi, and Hinglish, arrive at hundreds per day, and live siloed in the Play Store with no structure.

Without a review intelligence pipeline, the team is forced to:
•	Manually skim reviews — slow, biased, and not scalable
•	React to issues weeks late instead of spotting them in real time
•	Miss regional language signals that English-only tools ignore
•	Write weekly stakeholder updates from gut feel, not data

Core Problem: The product team lacks an automated pipeline that scrapes Play Store reviews, clusters them into actionable themes, generates a weekly pulse report via LLM, and delivers it to stakeholders — without any manual effort.

1.1  Why Now?
•	ChatGPT India app downloads crossed 50M — review volume is too high for manual reading
•	Voice feature was recently launched — early signal on pain points is critical
•	Groq's free-tier Llama 3 API makes LLM-powered analysis accessible at zero cost
•	GitHub Actions enables zero-infrastructure weekly automation

1.2  Success Looks Like
•	Every Monday morning: a one-page pulse report lands in your inbox — automatically
•	Top 5 themes identified from last week's reviews with review counts and sentiment
•	3 real user quotes (no PII) that illustrate each top theme
•	3 concrete product action items generated by the LLM
•	Full pipeline re-runs in under 5 minutes on any machine with one command

2. System Architecture
The pipeline is a linear 5-stage data flow. Each stage has a single responsibility, clean inputs/outputs, and can be re-run independently.

Pipeline Flow: Play Store → Scraper → Raw CSV → BERTopic Clusterer → Clustered CSV → Groq LLM Analyser → pulse.json → Email Sender → Inbox

2.1  Component Map
Component	Responsibility
scraper.py	Fetches latest 200 reviews from Play Store India (google-play-scraper)
cluster.py	Embeds reviews with sentence-transformers, runs BERTopic, assigns 5 themes
analyse.py	Sends themed reviews to Groq Llama 3, returns structured pulse JSON
report.py	Renders pulse.json into Markdown + HTML one-pager
emailer.py	Sends HTML email via Gmail SMTP using env-var credentials
main.py	Orchestrator — runs all 5 stages in sequence, handles errors
.github/workflows/weekly.yml	GitHub Actions cron: runs every Monday 07:00 IST
.env / GitHub Secrets	Stores GROQ_API_KEY, GMAIL_USER, GMAIL_APP_PASSWORD

3. Phase-wise Build Guide
Build in 5 phases. Each phase produces a working, testable artifact before you move on. Do not skip ahead.

Phase 1 — Scraper
Goal: Fetch 200 real Play Store reviews for com.openai.chatgpt, India, English. Save as raw_reviews.csv with no PII.
Inputs: App ID: com.openai.chatgpt | Country: in | Language: en
Outputs: raw_reviews.csv  (date, rating, title, review_text, platform)
Tools / Libs: google-play-scraper (npm)  |  Node.js 18  |  csv-writer
Key Prompts / Commands: node scraper.js  →  should print: Saved 200 reviews to raw_reviews.csv

Phase 2 — BERTopic Clustering
Goal: Load raw_reviews.csv, embed with sentence-transformers, run BERTopic, force exactly 5 themes. Save clustered CSV.
Inputs: raw_reviews.csv
Outputs: clustered_reviews.csv  (adds: theme_id 0-4, theme_label)
Tools / Libs: bertopic  |  sentence-transformers  |  scikit-learn  |  pandas
Key Prompts / Commands: python cluster.py  →  should print a table: Theme 0: N reviews | Theme 1: N reviews ...

Phase 3 — Groq LLM Analysis
Goal: For each of the 5 themes, send 10 representative reviews to Groq Llama 3. Ask it to return a structured JSON with label, summary, top 3 quotes, and 1 action item.
Inputs: clustered_reviews.csv  |  GROQ_API_KEY in .env
Outputs: pulse.json  —  {top_themes[], quotes[], action_items[], executive_summary}
Tools / Libs: groq  |  python-dotenv  |  json
Key Prompts / Commands: System prompt: "You are a product analyst. Return ONLY valid JSON. No markdown."  |  User prompt: "Here are {n} app reviews grouped under theme '{label}'. Extract: theme_name, summary (2 sentences), top_3_quotes (no usernames), action_item."

Phase 4 — Report Generation
Goal: Render pulse.json into a clean Markdown one-pager (<=250 words) and an HTML email-ready version.
Inputs: pulse.json  |  templates/pulse.md.j2  |  templates/pulse.html.j2
Outputs: weekly_pulse.md  |  weekly_pulse.html
Tools / Libs: Jinja2  |  markdown  |  python-dateutil
Key Prompts / Commands: python report.py  →  opens weekly_pulse.html in browser for preview

Phase 5 — Email Automation
Goal: Send weekly_pulse.html as an HTML email via Gmail SMTP. Schedule via GitHub Actions every Monday.
Inputs: weekly_pulse.html  |  GMAIL_USER + GMAIL_APP_PASSWORD in .env / GitHub Secrets
Outputs: Email in inbox  |  .github/workflows/weekly.yml
Tools / Libs: smtplib  |  email.mime  |  GitHub Actions
Key Prompts / Commands: python emailer.py  →  check inbox  |  GitHub Actions cron: "30 1 * * 1"  (01:30 UTC = 07:00 IST Monday)


4. LLM Prompt Templates
These are the exact prompts to use in Phase 3 with Groq. Copy them into analyse.py.

4.1  System Prompt
You are a senior product analyst at an AI company.
You analyse app store reviews and extract product insights.
Rules:
- Return ONLY valid JSON. No markdown, no explanation.
- Never include usernames, emails, or any PII in quotes.
- Quotes must be verbatim from the reviews provided.
- Be concise. Each summary must be 2 sentences max.

4.2  User Prompt (per theme)
Here are {n} Play Store reviews for the ChatGPT app (India),
grouped under the theme: '{theme_label}'.

REVIEWS:
{reviews_text}

Return a JSON object with exactly these keys:
{
  "theme_name": string,       // 3-5 word label
  "review_count": number,
  "sentiment": "positive"|"negative"|"mixed",
  "summary": string,          // 2 sentences
  "top_quotes": [string, string, string],  // no PII
  "action_item": string       // 1 concrete PM action
}

4.3  Final Summary Prompt
Given these 5 theme analyses for the ChatGPT India app:
{all_themes_json}

Return a JSON object:
{
  "executive_summary": string,  // 3 sentences, stakeholder-ready
  "top_3_themes": [string],     // theme names ranked by urgency
  "top_3_actions": [string],    // concrete product actions
  "week": "YYYY-WW"
}

6. GitHub Actions — weekly.yml
Place this file at .github/workflows/weekly.yml in your repo. Add GROQ_API_KEY, GMAIL_USER, and GMAIL_APP_PASSWORD as encrypted repository secrets.

name: Weekly Review Pulse

on:
  schedule:
    - cron: "30 1 * * 1"   # 01:30 UTC = 07:00 IST every Monday
  workflow_dispatch:        # allow manual trigger

jobs:
  pulse:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node 18
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          npm install
          pip install -r requirements.txt

      - name: Run pipeline
        env:
          GROQ_API_KEY: ${{ secrets.GROQ_API_KEY }}
          GMAIL_USER: ${{ secrets.GMAIL_USER }}
          GMAIL_APP_PASSWORD: ${{ secrets.GMAIL_APP_PASSWORD }}
        run: python main.py

      - name: Upload pulse report as artifact
        uses: actions/upload-artifact@v4
        with:
          name: weekly-pulse
          path: output/

8. README.md Template
Include this structure in your README.md:

# ChatGPT India — App Review Insights Analyser

Automated weekly pipeline: Play Store reviews → BERTopic clusters → Groq LLM pulse → Gmail.

## Setup
```bash
npm install
pip install -r requirements.txt
cp .env.example .env   # fill in your keys
```

## Run
```bash
python main.py         # runs all 5 phases
python main.py --phase scrape    # run one phase only
```

## Re-run for a new week
Just run python main.py again — it always fetches the latest reviews.

## Theme Legend
Themes are re-generated each week by BERTopic + LLM.
Typical themes for ChatGPT India: Voice UX, Language Support,
Paywall Friction, Response Quality, App Performance.

## Secrets required
GROQ_API_KEY    — from console.groq.com (free)
GMAIL_USER      — your Gmail address
GMAIL_APP_PASSWORD — 16-char App Password (not your Gmail password)

## GitHub Actions
Runs every Monday 07:00 IST automatically.
Manual trigger: Actions tab → Weekly Review Pulse → Run workflow.
